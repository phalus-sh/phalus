left-pad is a minimal npm package that provides a single utility function for padding a string (or value coercible to a string) on the left side with a specified character until the string reaches a desired total length. It is primarily used for text alignment, formatting output in terminals or logs, and zero-padding numbers. The package became historically notable in the npm ecosystem after its 2016 unpublishing incident caused widespread build failures, making it a reference case for dependency management discussions. Its scope is intentionally narrow: one function, one purpose — left-padding strings.
